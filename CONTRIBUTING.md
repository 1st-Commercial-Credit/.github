# Contributing

## PR rules (baseline)
- Small, reviewable PRs
- Tests updated/added
- Security: no secrets, no PII, no credentials committed

## Branching
- `main` is protected
- Feature branches: `feat/<short>`, `fix/<short>`, `chore/<short>`

## Commit hygiene
- Prefer Conventional Commits: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`
